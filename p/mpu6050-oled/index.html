MPU6050的I2C模式选择

  MPU6050 是独立的外部传感器芯片，不属于单片机片上外设；若要读取该外设内部寄存器数据，主控需要通过总线与该芯片建立通信。单片机与外部芯片常用通信协议包含 I2C、SPI、UART、CAN 等，本章将以 MPU6050 为例，讲解 I2C 通信原理与实操。

软件I2C

  软件 I2C 通过代码手动控制 SCL、SDA 两根通信引脚的高低电平，模拟标准 I2C 协议时序，实现主控与外部传感器芯片通信。 软件 I2C 优势是引脚选择灵活，绝大部分通用 GPIO 引脚，只要支持开漏输出配置，搭配内部上拉或外部上拉电阻，即可实现 I2C 总线通信；软件 I2C 必须严格遵循 I2C 规定时序，具体时序规范可查阅 I2C 官方手册。
  经过实测，HAL库软件模拟进行通信的效率比标准库低，主要是因为在与标准库同样配置下，HAL库手动翻转电平的效率比较耗时。

  

CubeMX 选项 标准库宏 官方最大翻转频率
Low GPIO_Speed_2MHz 2 MHz
Medium GPIO_Speed_10MHz 10 MHz
High GPIO_Speed_50MHz 50 MHz

  针对于软件I2C，高速PWM等，都推荐最高档！而普通 LED、继电器、低速 IO 输出则选择Low ， Medium即可。另外提一嘴，这里所说的最大速度是指 引脚电平切换，电压爬升或跌落的快慢，而不是代码多久翻转一次。GPIO 最大输出速度只控制 I2C 总线拉低电平的快慢，速度档位过低会导致下降沿过慢，I2C 时序不达标，出现通信失败；软件 I2C 场景建议固定配置 High 档位。
  而对于该平衡车项目，我使用软件I2C进行MPU6050数据获取时（也就是函数MPU6050_GetData(&MPU6050_Data);），经实测发现他大约耗时578us，而同样配置下，标准库仅耗时约352us。这也就让我开始尝试使用硬件I2C进行获取数据了。

硬件I2C

  硬件I2C，是指应用单片机为我们提供的片上外设,在CubeMx对应I2C模式，而后进行I2C通信，这里直接调用官方的函数接口就可以完成一系列操作，并不需要我们手动翻转电平来产生起始、终止信号等等。

标准I2C

  硬件I2C又分为标准模式和快速模式。标准模式下，I2C的时钟频率仅100KHZ，与MPU6050通信速度反而比软件I2C慢。个人在实验中，测出的硬件I2C的MPU6050数据获取耗时1594，1595us，这和软件I2C耗时578us，差了不止一个等级。不仅达不到我的目的——提高MPU6050获取数据的效率，反而还拖了后腿（当然，这里我都是基于HAL库版本进行讨论的）！因而这也就引出了后续选取硬件I2C的快速模式作为I2C通信模式。
  在引入快速I2C时，先来解释一下这里的I2C Clock Speed (Hz)。I2C 时钟频率，指I2C 总线 SCL 时钟信号的方波频率，代表每秒在 SCL 线上完整高低电平切换的次数，直接决定数据传输快慢。 100kHz = SCL 每秒完整跳变 10 万次；400kHz = 每秒跳变 40 万次。而我们知道，SCL作为时钟信号线，间接影响了SDA的数据读取和数据写入，具体表现是：SCL低电平期间，允许主机写数据，高电平期间允许从机读数据。因此，时钟频率越高，I2C读取速率也越大。但值得注意的是，快速模式并不是所有外部芯片都支持的，具体得去查阅芯片手册！当前的MPU6050是支持的。

I2C 协议的数据有效性规则：

SCL 状态 SDA 线行为
低电平 允许改变 SDA（写数据）
高电平 SDA 必须保持稳定（读数据）

快速I2C


值得注意的是，快速I2C多了个选项，Fast Mode Duty Cycle

  一般的通信要求都是选择低电平时间/高电平时间为2：1的，但既然多出个16/9的选项，说明有些通信确实对高低电平的时间占比有特殊要求，如果选错了，可能会导致通信失败或者效率变低，具体需要自行查阅资料。
  以MPU6050为例子，其实选择2/1或者16/9，都是能进行通信的。我进行测试的时候，2/1的组别能将MPU6050数据获取速率提快至416，417us，而16/9为461，462us，所以这里建议选择2/1占比的。
  这样，MPU6050的数据获取速率比使用软件I2C快了160us左右，但是始终比标准库的软件I2C读取速率慢！这也尽我所能了，如若有别的方法能进一步提高读取速率，麻烦发邮箱告知我一声！QQ邮箱：3152067902@qq.com！   当然，这里没有采用DMA协助数据运输以及I2C的中断，个人经考量觉得意义不大，因而没有进行测试。而且个人觉得416us的速率对于平衡车还是够快的了，所以跳过了DMA协助数据运输以及I2C的中断的测试。各位可以自行尝试一下。后续等我博客开启评论功能，可于评论区告诉一声！
总结

  1. DMA不能提高 I2C 硬件最高通信速率（SCL 频率不变）；
  2. DMA 代替 CPU 搬运字节，消除 CPU 延迟造成的总线停顿，批量连续传输时实际有效吞吐更高、耗时更少；
  3. 核心定位：减负 CPU，不是提速总线硬件时序。

#include "MPU6050.h"

#define MPU6050_Address 0xD0 // HAL库使用8位地址（左对齐），0x68 << 1 = 0xD0

void MPU6050_Init(void)
{
    MPU6050_WriteData(MPU6050_PWR_MGMT_1, 0x01); // 退出睡眠模式

    MPU6050_WriteData(MPU6050_PWR_MGMT_2, 0x00);

    MPU6050_WriteData(MPU6050_SMPLRT_DIV, 0x00); // 数据输出最快

    // MPU6050_WriteData(MPU6050_CONFIG, 0x06);
    MPU6050_WriteData(MPU6050_CONFIG, 0x00); //  CONFIG = 0x06 使能了数字低通滤波器，带宽仅 5Hz，陀螺仪数据延迟约 19ms。小车倾斜 1°→等 19ms 才能读到→PID 输出时已经倒得更厉害了－这是之前"无法完全直立"的根本原因。改为 CONFIG = 0x00 ，延迟降到约 1ms，控制环路能实时响应

    MPU6050_WriteData(MPU6050_GYRO_CONFIG, 0x18);

    MPU6050_WriteData(MPU6050_ACCEL_CONFIG, 0x18);
}

void MPU6050_WriteData(uint8_t Address, uint8_t data)
{
    HAL_I2C_Mem_WriteData(MPU6050_Address, Address, &data, 1);
}

int8_t MPU6050_ReadData(uint8_t Address)
{
    uint8_t data = 0x00;

    HAL_I2C_Mem_ReadData(MPU6050_Address, Address, &data, 1); // 地址HAL库自动完成加1操作

    return data;
}

void MPU6050_ReadDatas(uint8_t Address, uint8_t *DataArr, uint8_t Count)
{
    HAL_I2C_Mem_ReadData(MPU6050_Address, Address, DataArr, Count);
}

void MPU6050_GetData0(int16_t *AccX, int16_t *AccY, int16_t *AccZ,
                      int16_t *GyroX, int16_t *GyroY, int16_t *GyroZ)
{
    uint8_t Data[14];

    MPU6050_ReadDatas(MPU6050_ACCEL_XOUT_H, Data, 14);

    *AccX = (Data[0] << 8) | Data[1];
    *AccY = (Data[2] << 8) | Data[3];
    *AccZ = (Data[4] << 8) | Data[5];

    *GyroX = (Data[8] << 8) | Data[9];
    *GyroY = (Data[10] << 8) | Data[11];
    *GyroZ = (Data[12] << 8) | Data[13];
}

void MPU6050_GetData(MPU6050_Data_t *MPU6050_Data)
{
    uint8_t DataArr[14];

    MPU6050_ReadDatas(MPU6050_ACCEL_XOUT_H, DataArr, 14);

    MPU6050_Data->Accel_X = (DataArr[0] << 8) | DataArr[1];
    MPU6050_Data->Accel_Y = (DataArr[2] << 8) | DataArr[3];
    MPU6050_Data->Accel_Z = (DataArr[4] << 8) | DataArr[5];

    MPU6050_Data->Gyro_X = (DataArr[8] << 8) | DataArr[9];
    MPU6050_Data->Gyro_Y = (DataArr[10] << 8) | DataArr[11];
    MPU6050_Data->Gyro_Z = (DataArr[12] << 8) | DataArr[13];
}

#include "S_I2C.h"

/************************************************************************************/
void SI2C_WriteSCL(GPIO_PinState PinState)
{
    HAL_GPIO_WritePin(MPU6050_SCL_GPIO_Port,MPU6050_SCL_Pin , PinState);
}

void SI2C_WriteSDA(GPIO_PinState PinState)
{
    HAL_GPIO_WritePin(MPU6050_SDA_GPIO_Port,MPU6050_SDA_Pin , PinState);
}

uint8_t SI2C_ReadSDA(void)
{
    return HAL_GPIO_ReadPin(MPU6050_SDA_GPIO_Port,MPU6050_SDA_Pin);
}
/************************************************************************************/

void I2C_Init(void)
{
    SI2C_WriteSCL(1);
    SI2C_WriteSDA(1);
}

void I2C_Start(void)
{
    //防止两线异常处于低电平
    SI2C_WriteSDA(1);//先拉高SDA,可以兼容起始条件和重复起始条件，防止因为先拉高SCL后拉高SDA导致产生终止条件
    SI2C_WriteSCL(1);
    //以上为个人添加，如有不妥可以删除
    SI2C_WriteSDA(0);
    SI2C_WriteSCL(0);
}

void I2C_Stop(void)
{
    //防止两线异常
    SI2C_WriteSDA(0);//为确保SDA能产生上升沿
    SI2C_WriteSCL(0);//江协没有这个
    //以上为个人添加，如有不妥可以删除
    SI2C_WriteSCL(1);
    SI2C_WriteSDA(1);
}

void I2C_SendByte(uint8_t data)
{
    for (uint8_t i = 0; i < 8; i++)
    {
        SI2C_WriteSDA(data & (0x80 >> i));
        SI2C_WriteSCL(1);//驱动时钟走个脉冲
        SI2C_WriteSCL(0);
    }
}

uint8_t I2C_ReadByte(void)
{
    uint8_t data = 0x00;

    SI2C_WriteSDA(1);//主机释放总线控制权

    for (uint8_t i = 0; i < 8; i++)
    {
        SI2C_WriteSCL(1);
        data |= SI2C_ReadSDA() ? (0X80 >> i) : (0X00);//是否需要延时，待后续实测
        SI2C_WriteSCL(0);
    }

    return data;
}

void I2C_SendAckBit(uint8_t AckBit)
{
    SI2C_WriteSDA(AckBit);

    SI2C_WriteSCL(1);

    SI2C_WriteSCL(0);
}

uint8_t I2C_ReadAckBit(void)
{
    uint8_t AckBit = 0x00;

    SI2C_WriteSDA(1);//主机释放总线控制权

    SI2C_WriteSCL(1);

    AckBit = SI2C_ReadSDA();//是否需要延时，待后续实测

    SI2C_WriteSCL(0);

    return AckBit;
}

#include "MPU6050.h"

#define MPU6050_WriteCommand 0xD0 // 与上读写位          不能直接发送地址0x68，要加上读写位
#define MPU6050_ReadCommand 0xD1  // 与上读写位

void MPU6050_Init(void)
{
    MPU6050_WriteData(MPU6050_PWR_MGMT_1, 0x01); // 退出睡眠模式

    MPU6050_WriteData(MPU6050_PWR_MGMT_2, 0x00);

    MPU6050_WriteData(MPU6050_SMPLRT_DIV, 0x00); // 数据输出最快

    // MPU6050_WriteData(MPU6050_CONFIG, 0x06);
    MPU6050_WriteData(MPU6050_CONFIG, 0x00); //  CONFIG = 0x06 使能了数字低通滤波器，带宽仅 5Hz，陀螺仪数据延迟约 19ms。小车倾斜 1°→等 19ms 才能读到→PID 输出时已经倒得更厉害了－这是之前"无法完全直立"的根本原因。改为 CONFIG = 0x00 ，延迟降到约 1ms，控制环路能实时响应

    MPU6050_WriteData(MPU6050_GYRO_CONFIG, 0x18);

    MPU6050_WriteData(MPU6050_ACCEL_CONFIG, 0x18);
}

void MPU6050_WriteData(uint8_t Address, uint8_t data)
{
    I2C_Start();

    I2C_SendByte(MPU6050_WriteCommand); // 发送器件写地址
    I2C_ReadAckBit();
    // 应答位先不管
    I2C_SendByte(Address); // 发送寄存器写地址
    I2C_ReadAckBit();
    // 应答位先不管
    I2C_SendByte(data); // 可以试一下指定地址写多个字节
    I2C_ReadAckBit();

    I2C_Stop();
}

int8_t MPU6050_ReadData(uint8_t Address)
{
    uint8_t data = 0x00;

    I2C_Start();

    I2C_SendByte(MPU6050_WriteCommand); // 发送器件写地址   因为要写寄存器地址，所以要先写
    I2C_ReadAckBit();
    // 应答位先不管
    I2C_SendByte(Address); // 发送寄存器地址
    I2C_ReadAckBit();
    // 应答位先不管
    I2C_Start();                       // 重复起始，转入读的时序，就必须重新指定读写位
    I2C_SendByte(MPU6050_ReadCommand); // 发送器件读地址
    I2C_ReadAckBit();

    data = I2C_ReadByte(); // 可以试一下指定地址读多个字节
    I2C_SendAckBit(1);     // 如果要读多个字节，则应该给应答（从机收到应答后会继续发送数据），但只读一个的话，给非应答就行

    I2C_Stop();

    return data;
}

void MPU6050_ReadDatas(uint8_t Address, uint8_t *DataArr, uint8_t Count)
{
    I2C_Start();

    I2C_SendByte(MPU6050_WriteCommand); // 发送器件写地址   因为要写寄存器地址，所以要先写
    I2C_ReadAckBit();
    // 应答位先不管
    I2C_SendByte(Address); // 发送寄存器地址
    I2C_ReadAckBit();
    // 应答位先不管
    I2C_Start();                       // 重复起始，转入读的时序，就必须重新指定读写位
    I2C_SendByte(MPU6050_ReadCommand); // 发送器件读地址
    I2C_ReadAckBit();
    for (uint8_t i = 0; i < Count; i++)
    {
        DataArr[i] = I2C_ReadByte(); // 可以试一下指定地址读多个字节
        if (i != Count - 1)
        {
            I2C_SendAckBit(0); // 如果要读多个字节，则应该给应答（从机收到应答后会继续发送数据），但只读一个的话，给非应答就行
        }
        else
        {
            I2C_SendAckBit(1);
        }
    }

    I2C_Stop();
}

void MPU6050_GetData0(int16_t *AccX, int16_t *AccY, int16_t *AccZ,
                      int16_t *GyroX, int16_t *GyroY, int16_t *GyroZ)
{
    uint8_t Data[14];

    MPU6050_ReadDatas(MPU6050_ACCEL_XOUT_H, Data, 14);

    *AccX = (Data[0] << 8) | Data[1];
    *AccY = (Data[2] << 8) | Data[3];
    *AccZ = (Data[4] << 8) | Data[5];

    *GyroX = (Data[8] << 8) | Data[9];
    *GyroY = (Data[10] << 8) | Data[11];
    *GyroZ = (Data[12] << 8) | Data[13];
}
